Git-Github

Git ble utviklet av Linus Thorvards som er også skaperen bak Linux. Git er et versjonskontrollsystem (VCS) som er et verktøy som gjør det mulig å samarbeide med flere personer på et felles prosjekt eller flere prosjekter samtidig.
VCS virker som et sted der du kan gjøre endringer på kode uten å påvirke selve master filen, dersom endringene er godkjent kan man enkelt opplaste de til masteren. Samtidig har man mulighet til å gå tilbake til eldre filer dersom det har blitt oppdaget noen feil.
Git er raskt og effektivt da nesten alt er lagret lokalt, samtidig så får alle som jobber på samme prosjekt en kopi av «repoet» Dette er en fil som oppdateres automatisk, hvor alle til enhver tid får oversikt over endringene og historien til filene.
REPOSITIORIES:
Oppbevaringsted for filene i Git Prosjekt.
BRANCHES:
En branch er uavhengig serie med commits i versjonsdatabasen.
COMMIT: Et objekt som holder info om en spesifikk endring.
En funksjon hvor det er mulig å gjøre endringer på kode uten å påvirke selve master filen, disse blir lagret i en separat mappe som gjør det enkelt å lage, endre eller slette kode uten å påvirke hovedmappen.
Git webtjenester: Det er ulike webapplikasjoner som gjør det enklere å jobbe med git, f.eks, GitLab, GitHub, Bitbucker.
Git hub er en open source plattform for utviklere der det er mulig å samarbeide med prosjekter uansett hvor man er.
Github bruker repositiories, branches og commits disse er funksjoner som gjør det mulig å gjøre endringer på filer uten å påvirke selve hovedfilen Master.
GitHub er webtjenesten til Git.

Hva synes jeg om github?

Jeg synes Github er et utrolig fint verktøy å bruke når man jobber med kode. Det kan være ofte jeg gjør endringer som jeg angrer i etterkant, som gjør at jeg må rote meg gjennom CTRL+Z for å gå tilbake.
Samtidig føler jeg at det gir en mye bedre oversikt som lar meg se hvor og når jeg har gjort endringer.
Har ikke testet denne i prosjekter før, men det er helt klart mitt første valg for når den tid kommer. Super uber bra tjeneste
